# what have we built right now ??

<ol start="3">
<li>RAG Pipeline (Retrieval-Augmented Generation)</li>
</ol>
<p>• Houses <strong>417 official bank product files</strong> (Fixed Deposits, Loans, Savings accounts, etc.). When a recommendation or chat query is made, it performs a search (<strong>BM25</strong>) on these files to fetch the latest guidelines and rates to feed to the AI.</p>
<ol start="4">
<li>Rules Engine</li>
</ol>
<p>• Performs exact mathematical scoring and projections (e.g. compound interest, savings ratios, debt metrics) in pure Python first, ensuring the AI receives mathematically correct values and does not hallucinate numbers.</p>

<p>The Agentic Tool Loop</p>
<p>When a user types a query like &quot;Am I saving enough for my house goal?&quot;, the AI does not make up an answer. It follows a multi-turn reasoning process:</p>
<p>[User Message] → &quot;Am I saving enough for my house goal?&quot;
|
▼
[Groq LLM Calls] (Step 1) → &quot;I need to inspect the user&#39;s profile and goal details.&quot;
|
├─ Tool Call 1: get_customer_profile() → Returns name, monthly savings, and active goals list.
|
▼
[Groq LLM Calls] (Step 2) → &quot;I see a Buy House goal. I need to calculate its trajectory.&quot;
|
├─ Tool Call 2: calculate_goal_projection(goal_name=&quot;Buy House&quot;) → Returns math projection details.
|
▼
[Groq LLM Calls] (Step 3) → &quot;Calculations show the user is ₹5,000/month behind. I will synthesize the advice.&quot;
|
▼
[Final Response] → &quot;You are currently behind on your Buy House goal by 4 months. I recommend increasing your monthly SIP by...&quot;</p>
<p>Available Tools:</p>
<ol>
<li>get_customer_profile: Fetches the active user&#39;s demographic and financial summaries.</li>
<li>get_transactions: Retrieves transaction records with optional filters (e.g. food categories, credits, debits).</li>
<li>calculate_goal_projection: Solves the goal compound timelines and outputs recommendations in Python.</li>
<li>search_idbi_knowledge: Queries the 417-document product RAG index for official interest rates, loans, cards, or policies.</li>
</ol>


# problem

<p>Problem</p>
<p>Most customers don&#39;t receive personalized investment advice because wealth managers can&#39;t serve everyone individually.</p>
<p>Goal</p>
<p>Create an AI-powered virtual wealth advisor (avatar/chatbot) integrated into the bank&#39;s mobile app.</p>
<p>Inputs</p>
<p>• Customer spending patterns
• Income
• Savings
• Existing investments
• Financial goals
• Risk profile</p>

# what info about customers can we get about user from idbi in reality?
<p>Things the bank almost certainly has (inferable/queryable):</p>
<p>• KYC data: age, declared income (at account opening), occupation, address/branch, PAN-linked info
• Account balance, transaction history (UPI, NEFT, debit/credit card spends) — categorizable by merchant/MCC codes
• Existing IDBI holdings: FDs, RDs, loans, credit cards, any MFs/insurance sold through IDBI
• Loan repayment history, EMI obligations
• Possibly: CIBIL/credit bureau pull (if IDBI has that integration) — this is how you&#39;d see holdings outside IDBI (the GM literally said this in the AMA: &quot;we will have access to credit bureau data&quot;)</p>
